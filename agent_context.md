## 3. Core Architecture

### Intelligence Layer (GraphRAG)
- **`src/app/api/chat/route.ts`**: The core AI endpoint. It uses the Vercel AI SDK to stream responses from Groq's `llama-3.1-8b-instant` (or 70B if updated). It has access to two main tools:
  1. `queryDatabase`: Takes a Cypher query generated by the LLM, runs it against the Neo4j DB using `src/lib/neo4j.ts`, and returns the exact stats (GraphRAG).
  2. `showPlayerGraphics`: A tool that triggers the frontend to un-hide the Advanced Tracking Visuals.
- **`src/components/PredictionEngine.tsx`**: The chat UI. It listens to the AI stream. If it detects a `toolInvocation` for `showPlayerGraphics`, it updates the frontend state to display the graphs.

### Visualization Layer
- **`src/app/page.tsx`**: The main dashboard layout. Holds state for `activePlayerGraphics`. If null, visuals are hidden to save bandwidth.
- **`src/components/TrackingDashboard.tsx`**: The parent component for all visuals.
- **2D Visuals**: `Beehive.tsx`, `PitchMap.tsx`, `WagonWheel.tsx`. These render SVGs based on `tracking_data.json`. The rendering logic has been overridden mathematically inside the components to ensure the visuals look strictly realistic (e.g., wickets hit the stumps, wagon wheel distances map to runs scored).
- **3D Visuals (`src/components/Pitch3D.tsx`)**: Uses React Three Fiber to render a 3D cricket pitch. `TrackingDashboard` maps the 2D ball coordinates into a 3D trajectory (Release point Z=-10, Bounce point mapped, Wicket point Z=10).

## 4. Database Setup (Neo4j)
- Connection utility is in `src/lib/neo4j.ts`.
- The database is pre-populated with a basic ontology (Players, Matches, Deliveries). 
- An ingestion script `ingest.mjs` exists in the root directory which was used to create the base nodes (Bumrah, Kohli, Starc, Rohit) and relationships (`BOWLED_TO`).
